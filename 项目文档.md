# 绩效考核管理系统

> 企业级多部门绩效考核与审核管理平台

**Vue 3 前端** · **Laravel API** · **MySQL 数据库** · **多部门架构** · **RBAC 权限**

---

## 📋 项目概述

**绩效考核管理系统**是一套面向企业内部多部门的绩效考核管理平台，覆盖 **人事部、经纪部、中转部、运营部、主播部、财务部、总经办**共 **7 个部门**，支持部门总监、组长、员工 **3 级角色**的权限管控与工作流协作。

该系统采用**前后端分离架构**，前端基于 uni-app (Vue 3) 构建 H5 应用，后端基于 Laravel 提供 RESTful API，支持考核发布、任务提交、三级审核流程、图片凭证上传、数据统计仪表盘等完整业务闭环。独立完成全栈开发，从需求分析到上线部署全过程独立交付。

## ⚙️ 核心功能模块

### 📝 考核发布管理

- HR 人事部统一发布各周期考核任务
- 按部门设定差异化考核指标（经纪部：招募量；主播部：直播天数/时长/流水；运营部：流水目标等）
- 支持考核周期管理、目标备注、规则说明
- 考核生命周期：**待提交 → 进行中 → 到期完结**

### 📤 员工考核提交

- 各部门专属提交表单（主播部：直播数据；经纪部：招募数量等）
- 支持图片凭证上传（压缩、预览、进度提示）
- 已上传未提交的图片自动回显，防止重复上传
- 资源明细录入（平台、联系方式等结构化数据）
- 提交后进入审核流程，状态实时同步

### 🔍 三级审核机制

- 员工提交 → **财务部初审**
- 财务初审通过 → **总经办终审**
- 审核结果：达标 / 不达标，附审核意见
- 人事部可全流程监控各周期考核状态

### 📊 仪表盘与统计

- 各部门总监端仪表盘：待提交清单、进度概览
- 人事考核总览：按周期/部门查看完成率、达标率
- KPI 指环图可视化展示达标、未达标、待审核比例
- 历史考核记录查询与回溯

### 🔐 RBAC 权限控制

- 3 级角色：总监 / 组长 / 员工，各有独立路由映射
- 登录时根据 dept+role 自动路由到对应工作台
- 仅审核部门（财务、总经办）无自主考核权
- API 中间件 Token 拦截，所有请求携带 Bearer 鉴权

### 🔧 扩展功能

- 人事部：人员账号 CRUD、劝退记录台账
- 中转部：主播流转管理、入职转正流程
- 运营部：每日流水提交、流水审核、台账管理
- 兼职主播管理模块
- 考核文档 Word 导出

## ✨ 技术亮点

### 前后端分离架构
uni-app H5 前端 + Laravel RESTful API，通过 Vite 代理开发、独立构建部署。Token 鉴权机制，支持跨域请求与状态持久化。

### RBAC 多角色路由
基于部门 ID + 角色值的二维路由映射表，登录后自动分发到对应工作台。7 部门 × 3 角色的精细化权限管控。

### 图片凭证防重复上传
图片先上传入库（assess_images），提交时才关联考核记录。离开页面不丢失，下次进入自动回显已上传图片，避免重复提交冗余凭证。

### 自定义设计系统
纯手写 CSS 组件库，包括卡片、进度条、状态标签、表单、Toast、骨架屏等 50+ 可复用组件，统一的色彩体系和间距规范，无第三方 UI 库依赖。

### 差异化部门指标
每个部门的考核指标表独立设计，通过 dept_id 驱动差异化表单渲染和数据持久化。主播部支持多维指标（天数/时长/流水/大哥数），经纪部支持招募量等。

### 数据库幂等迁移
使用 information_schema 检测字段是否存在后再 ALTER TABLE，所有迁移脚本可重复执行不报错，兼容 MySQL 5.5 低版本环境。
